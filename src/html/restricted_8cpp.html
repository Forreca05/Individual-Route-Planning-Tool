Detailed Description

Implementation of the restricted route algorithm.

Computes the best route with resctictions as: Avoiding specific nodes, Avoiding specific segments of the graph, Simultaniously avoiding nodes and sections, Including specific nodes the route has to visit.

Runs the restricted pathfinding algorithm, considering avoidance and inclusion constraints.

This function first checks if a specific node must be included in the path. If so, it calculates the shortest path to that node first and then continues from it to the final destination. If no inclusion is required, a direct shortest path is computed.

Parameters
graphThe graph where to search.
sourceThe starting node.
destinationThe target node.
includeNodeA node that must be included in the path (-1 if not applicable).
avoidNodesSet of nodes to be avoided.
modeThe mode of transportation.
parkedIndicates if there is parking avaliable.
outThe output stream to write the result.

Complexity is O((V + E) log V), where V is the number of vertices and E is the number of edges in the graph.
