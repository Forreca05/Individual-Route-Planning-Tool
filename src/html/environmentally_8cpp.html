Detailed Description

Implentation of the algorithm that computes the best route with both walking and driving.

Computes routes that combine driving and walking, allowing users to: Drive the first section of the route and then park the vehicle. Walk the remaining distance to the destination. There is also a max walking time. ( Time the user is willing to walk after parking.)

Runs the environmentally friendly route algorithm.

This function calculates a route that minimizes driving distance while ensuring the walking distance from a parking node to the destination does not exceed a given maximum.

Parameters
graphThe graph where to search.
sourceThe starting node.
destinationThe target node.
maxWalkThe maximum walking distance the user is willing to walk.
avoidNodesSet of nodes to be avoided.
time_to_idMap of parking nodes to their respective driving times.
modeThe mode of transportation (only "driving-walking" is valid).
parkedIndicates of there is parking avaliable.
outThe output stream to write the result.

Complexity is O((V + E) log V), where V is the number of vertices and E is the number of edges, since Dijkstra's algorithm is used multiple times. [O(n log(n))]
